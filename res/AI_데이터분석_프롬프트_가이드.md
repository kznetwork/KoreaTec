## 🎯 1일차 실습 프롬프트

### 세션 2: 프롬프트 엔지니어링 실습 (10:00-11:00)

#### 실습 1-1: 기본 프롬프트 연습
```
첨부한 defect_prediction_data.csv 파일의 기본 정보를 알려줘.
몇 개의 데이터가 있고, 어떤 컬럼들이 있는지 설명해줘.
```

**기대 결과:** 데이터 개수(2,000개), 컬럼 목록, 데이터 타입 확인

---

#### 실습 1-2: 구체적 질문 연습
```
첨부한 불량 예측 데이터에서:
1. 전체 불량률이 몇 %인지 계산해줘
2. 불량(1)과 정상(0)의 개수를 각각 알려줘
3. 이 불량률이 높은 편인지 낮은 편인지 제조업 기준으로 판단해줘
```

**기대 결과:** 불량률 37.6%, 불량 752개, 정상 1,248개, 일반적으로 높은 편

---

#### 실습 1-3: 비교 분석 프롬프트
```
다음 두 프롬프트의 차이를 비교해줘:

나쁜 예: "데이터 분석해줘"
좋은 예: "온도가 80도 이상일 때와 80도 미만일 때의 불량률을 각각 계산하고, 
         어느 쪽이 불량률이 더 높은지 비교해줘"

어떤 점이 다르고, 왜 좋은 예가 더 나은지 설명해줘.
```

### 세션 4: AI 기반 기초 통계 (13:00-14:30)

#### 실습 4-1: ChatGPT를 이용한 시장조사

```

```
첨부한 제조 공정 데이터를 읽고, 데이터의 전체 구조를 파악해서 요약해주세요.
```

```
첨부한 defect_prediction_data.csv 파일을 읽고, 데이터의 전체 구조를 분석해주세요.

1. 전체 행과 열의 개수를 알려주세요
2. 각 컬럼의 데이터 타입과 의미를 파악해주세요
3. '불량' 컬럼의 분포를 확인하고
   불량률(불량 비율)을 계산해주세요
4. 결측치가 있는지 확인해주세요

결과를 표로 정리해주세요.
```

```
주요 수치형 변수들의 기술통계량을 표로 보여주고, 히스토그램과 박스플롯으로 시각화해주세요.
```

```
"수치형 변수들(온도, 압력, 속도, 진동, 설비연식)의 기초 통계량을 분석해주세요.

1. 각 변수의 평균, 표준편차, 최소값, 최대값,
   사분위수를 계산해서 표로 보여주세요
2. 히스토그램으로 각 변수의 분포를 시각화해주세요
3. 박스플롯으로 이상치를 탐지해주세요
4. 어떤 변수에 이상치가 많은지 정리해주세요"
```

#### 실습 4-1: 기초 통계량 계산
```
첨부한 데이터에서 수치형 변수들(온도, 압력, 속도, 진동, 설비연식)의 
다음 통계량을 계산해줘:
1. 평균값
2. 표준편차
3. 최솟값과 최댓값
4. 중앙값

그리고 각 통계량이 제조 현장에서 어떤 의미인지 간단히 설명해줘.
```

---

#### 실습 4-2: 정규화 이해하기
```
온도 데이터를 0~1 범위로 정규화하는 과정을 설명해줘.
- 현재 온도의 최솟값과 최댓값을 먼저 알려주고
- 정규화 공식을 설명하고
- 예시로 온도 85도가 정규화되면 어떤 값이 되는지 계산해줘
```

---

#### 실습 4-3: 표준편차 활용
```
온도 데이터의 평균과 표준편차를 이용해서:
1. 평균 ± 1 표준편차 범위를 계산해줘
2. 이 범위를 벗어나는 데이터가 몇 개인지 세어줘
3. 이런 데이터를 어떻게 해석해야 하는지 설명해줘
```

---

### 세션 5: 확률 분포와 표본 (14:30-16:00)

#### 실습 5-1: 분포 이해하기
```
온도 데이터의 분포를 분석해줘:
1. 온도가 60~70도 사이인 데이터가 몇 개인지
2. 온도가 70~80도 사이인 데이터가 몇 개인지
3. 온도가 80~90도 사이인 데이터가 몇 개인지
4. 온도가 90도 이상인 데이터가 몇 개인지

각 구간별로 개수와 전체 대비 비율(%)을 알려줘.
```

---

#### 실습 5-2: 샘플링 연습
```
전체 2,000개 데이터에서 무작위로 100개를 샘플링해서:
1. 샘플의 평균 온도를 계산해줘
2. 전체 데이터의 평균 온도와 비교해줘
3. 차이가 있다면 왜 그런지 설명해줘
4. 몇 개의 샘플이 대표성을 가지려면 적절한지 의견을 줘
```

#### 세션 4: Step 6. 가설 검정
```
1. 각 변수에 대해 독립표본 t-검정을 수행해주세요
   (귀무가설: 두 그룹의 평균이 같다)
   
2. t-통계량, p-value, 효과 크기(Cohen's d)를 계산해주세요
   
3. p-value < 0.05인 변수를 찾아주세요

4. 결과를 표로 정리하고, 비즈니스 관점에서 해석해주세요"
```

### 세션 6: 현장 데이터 분석 기초 (16:00-17:30)

#### 실습 6-1: 데이터 탐색 (EDA)
```
첨부한 불량 예측 데이터를 탐색해줘:
1. 각 컬럼별로 어떤 특징이 있는지 요약해줘
2. 결측치(빈 값)가 있는지 확인해줘
3. 범주형 변수(원료등급, 교대조, 요일)의 고유값들을 알려줘
4. 가장 흥미로운 패턴이나 특이점을 3가지 찾아줘
```

---

#### 실습 6-2: 시각화 요청
```
다음 그래프를 만들어줘:
1. 온도와 불량 여부의 관계를 보여주는 박스플롯
2. 압력과 불량 여부의 관계를 보여주는 박스플롯
3. 불량 데이터와 정상 데이터의 온도 분포 비교 히스토그램

각 그래프에서 발견되는 패턴을 설명해줘.
```

---

#### 실습 6-3: 상관 분석
```
수치형 변수들(온도, 압력, 속도, 진동, 설비연식) 간의 상관관계를 분석해줘:
1. 상관계수 행렬을 만들어줘
2. 불량과 가장 강한 상관관계를 보이는 변수 3가지를 찾아줘
3. 이 변수들이 불량에 어떻게 영향을 미치는지 설명해줘
```

---

#### 실습 6-4: 회귀 분석 입문
```
온도를 독립변수로, 불량 여부를 종속변수로 하는 간단한 분석을 해줘:
1. 온도가 높을수록 불량률이 높아지는지 확인해줘
2. 온도를 10도 단위로 나눠서 각 구간의 불량률을 계산해줘
3. 결과를 표로 정리하고, 어느 온도 구간이 가장 위험한지 알려줘
```

---

#### 실습 6-5: 복합 조건 분석
```
다음 조건들을 동시에 만족할 때의 불량률을 분석해줘:
1. 온도 85도 이상 + 원료등급 C
2. 설비연식 8년 이상 + 야간 교대조
3. 압력 160 이상 + 속도 110 이상

각 조건 조합별 불량률과 해당 데이터 개수를 알려주고,
어떤 조합이 가장 위험한지 순위를 매겨줘.
```

## 🎯 2일차 실습 프롬프트

### 세션 2: 데이터 정제 및 이상치 제거 (10:30-12:00)

#### 실습 2-1: 결측치 확인
```
데이터에 결측치가 있는지 확인해줘:
1. 각 컬럼별로 결측치 개수를 세어줘
2. 만약 결측치가 있다면 어떻게 처리하는 것이 좋을지 3가지 방법을 제안해줘
3. 각 방법의 장단점을 설명해줘
```

---

#### 실습 2-2: 이상치 탐지
```
온도 데이터에서 이상치를 찾아줘:
1. IQR(사분위수 범위) 방법을 사용해서 이상치 기준을 계산해줘
2. 이 기준에 따라 이상치로 판단되는 데이터가 몇 개인지 세어줘
3. 이상치 데이터의 실제 값들을 보여줘
4. 이 값들이 정말 이상한지, 아니면 정상적인 극값인지 판단해줘
```

---

#### 실습 2-3: 이상치 제거 실습
```
다음 기준으로 이상치를 제거한 새로운 데이터를 만들어줘:
1. 온도가 평균 ± 3 표준편차를 벗어나는 데이터
2. 압력이 평균 ± 3 표준편차를 벗어나는 데이터

이상치 제거 전후의 데이터 개수와 불량률 변화를 비교해줘.
```

---

#### 실습 2-4: 중복 데이터 확인
```
중복된 데이터가 있는지 확인해줘:
1. 생산번호가 중복된 경우가 있는지 체크해줘
2. 모든 컬럼이 동일한 중복 행이 있는지 체크해줘
3. 만약 중복이 있다면 제거 후 데이터 개수를 알려줘
```

---

### 세션 3: AI 학습 데이터 변환 (13:00-14:30)

#### 실습 3-1: 범주형 변수 인코딩
```
범주형 변수를 숫자로 변환해줘:
1. 원료등급 (A/B/C) → 숫자로 변환
2. 교대조 (주간/야간) → 0/1로 변환
3. 요일 → 숫자로 변환 (월=1, 화=2, ...)

각 변환에 대해 어떤 방식을 사용했는지 설명해줘.
```

---

#### 실습 3-2: 정규화 실습
```
수치형 변수들을 0~1 범위로 정규화해줘:
1. 온도, 압력, 속도, 진동, 설비연식을 각각 정규화
2. 정규화 전후의 최솟값, 최댓값, 평균을 비교해줘
3. 정규화가 왜 필요한지 예시와 함께 설명해줘
```

---

#### 실습 3-3: 데이터 분할
```
데이터를 학습용과 검증용으로 나눠줘:
1. 전체 데이터의 80%는 학습용, 20%는 검증용으로 분할
2. 각 그룹의 데이터 개수와 불량률을 알려줘
3. 두 그룹의 불량률이 비슷한지 확인해줘
4. 왜 데이터를 나누는지, 비율은 왜 80:20인지 설명해줘
```

---

#### 실습 3-4: 변환 검증
```
데이터 변환이 제대로 되었는지 검증해줘:
1. 정규화된 값들이 실제로 0~1 범위 안에 있는지 확인
2. 범주형 변수가 숫자로 잘 변환되었는지 확인
3. 변환 과정에서 데이터 손실이 없는지 확인 (행 개수 비교)
4. 변환 전후 불량률이 동일한지 확인
```

### 세션 4: 불량률 예측 및 원인 분석 (14:30-16:00)

#### 실습 4-1: 주요 변수 분석
```
불량에 가장 큰 영향을 미치는 변수를 찾아줘:
1. 각 수치형 변수와 불량의 상관계수를 계산해줘
2. 상관계수가 높은 순서대로 변수를 정렬해줘
3. 상위 3개 변수를 선택하고, 왜 이 변수들이 중요한지 설명해줘
```

---

#### 실습 4-2: 조건별 불량률 분석
```
다음 조건들에서 불량률을 계산해줘:

온도 기준:
- 60도 미만
- 60~70도
- 70~80도
- 80~90도
- 90도 이상

설비연식 기준:
- 3년 미만
- 3~5년
- 5~7년
- 7년 이상

각 구간별 불량률과 데이터 개수를 표로 정리하고,
가장 위험한 구간을 찾아줘.
```

---

#### 실습 4-3: 복합 원인 분석
```
불량률이 50% 이상인 조건 조합을 찾아줘:
1. 2개 변수의 조합으로 불량률 50% 이상인 경우
2. 3개 변수의 조합으로 불량률 50% 이상인 경우

상위 5개 조합을 불량률이 높은 순서로 정렬하고,
각 조합의 불량률, 데이터 개수, 전체 대비 비율을 알려줘.
```

---

#### 실습 4-4: 개선 방안 도출
```
분석 결과를 바탕으로 불량률을 줄이기 위한 조치를 제안해줘:
1. 가장 시급하게 개선해야 할 공정 조건 3가지
2. 각 조건을 개선하면 예상되는 불량률 감소 효과
3. 현실적으로 적용 가능한 개선 방안 (온도 조절, 설비 교체 등)
4. 개선의 우선순위와 그 이유
```

---

#### 실습 4-5: 예측 모델 해석
```
만약 간단한 예측 모델을 만든다면:
1. 어떤 변수들을 입력으로 사용할지 선택해줘 (최대 5개)
2. 선택한 이유를 설명해줘
3. 이 변수들만으로 불량을 얼마나 정확하게 예측할 수 있을지 추정해줘
4. 예측 정확도를 높이려면 어떤 추가 정보가 필요한지 제안해줘
```

### 세션 5: 종합 실습 시나리오 (16:00-17:30)

#### 종합 실습: 전체 프로세스 실행
```
다음 시나리오를 처음부터 끝까지 수행해줘:

[상황]
공장장이 불량률이 너무 높다며 원인 분석을 요청했습니다.
당신은 데이터 분석 담당자로서 다음 작업을 수행해야 합니다.

[요구사항]
1. 데이터 품질 확인 (결측치, 이상치, 중복)
2. 필요시 데이터 정제
3. 주요 불량 원인 3가지 도출
4. 각 원인의 영향도 분석 (데이터와 통계로 증명)
5. 개선 방안 제시 (구체적인 조치와 예상 효과)
6. 최종 보고서 형태로 정리

보고서는 다음 형식으로 작성해줘:
- 요약 (3줄 이내)
- 데이터 분석 결과 (표와 수치 포함)
- 주요 발견사항 (불량 원인 TOP 3)
- 개선 권고안 (우선순위별)
- 결론
```

#### 심화 실습: 가상 시나리오 분석
```
다음 가상 시나리오에서 불량률을 예측해줘:

새로운 생산 계획:
- 온도: 82도

2. 가장 유사한 과거 데이터 5개를 찾아 그 결과를 보여줘
3. 위험 요소가 있다면 어떤 변수를 조정하면 좋을지 제안해줘
4. 조정 후 예상 불량률을 계산해줘
```

## 💡 프롬프트 작성 팁

### ✅ 좋은 프롬프트의 특징

1. **명확하고 구체적**
   - ❌ "데이터 분석해줘"
   - ✅ "온도가 85도 이상일 때의 불량률을 계산하고, 전체 평균 불량률과 비교해줘"

2. **단계별 요청**
   - ❌ "불량 원인 찾아줘"
   - ✅ "1) 각 변수별 불량률 계산, 2) 상위 3개 변수 선정, 3) 이유 설명"

3. **맥락 제공**
   - ❌ "그래프 그려줘"
   - ✅ "온도와 불량의 관계를 보여주는 산점도를 그리고, 패턴을 설명해줘"

4. **출력 형식 지정**
   - ❌ "통계 알려줘"
   - ✅ "평균, 표준편차, 최소/최대값을 표 형식으로 정리해줘"

---

### ❌ 피해야 할 프롬프트

1. **너무 모호한 질문**
   - "이 데이터 좋아?"
   - "분석 좀 해줘"

2. **지나치게 복잡한 요청**
   - 한 번에 10가지 이상의 작업 요청
   - 서로 관련 없는 여러 분석을 동시에 요청

3. **불가능한 요청**
   - "100% 정확한 예측 모델 만들어줘"
   - "미래 데이터를 생성해줘"

### 🎯 효과적인 프롬프트 패턴

#### 패턴 1: 탐색형
```
[데이터/변수]에 대해:
1. 기본 정보 확인
2. 분포 파악
3. 특이점 찾기
```

#### 패턴 2: 비교형
```
[조건 A]일 때와 [조건 B]일 때의 [결과]를 비교해서:
1. 각각의 값 계산
2. 차이 분석
3. 이유 설명
```

#### 패턴 3: 원인 분석형
```
[결과]에 영향을 미치는 요인을:
1. 전체 변수에서 찾기
2. 중요도 순위 매기기
3. 상위 N개 선정하고 이유 설명
```

#### 패턴 4: 의사결정 지원형
```
[상황]에서 [목표]를 달성하려면:
1. 현재 상태 분석
2. 개선 방안 제시
3. 예상 효과 계산
4. 우선순위 제안
```
